# Phase 8: RDS PostgreSQL and Secrets Manager

## What We Are Doing

We are adding a managed PostgreSQL database and giving ECS services the database connection string through AWS Secrets Manager.

## Why We Are Doing It

Applications need persistent data, but container tasks are temporary. If an ECS task stops and starts again, any local container data disappears. RDS gives the app a managed database that survives task restarts and deployments.

Secrets Manager keeps credentials out of source code, Docker images, and Terraform variables.

## Resources Added

- RDS PostgreSQL instance
- database security group
- DB subnet group
- generated database password
- Secrets Manager secret containing `DATABASE_URL`
- IAM permission for ECS task execution role to read the secret
- ECS task definition updates for `flight-service`, `booking-service`, and `checkin-service`

## Security Flow

- ALB can reach ECS task ports.
- ECS tasks can reach PostgreSQL on port `5432`.
- The database is not publicly accessible.
- ECS reads `DATABASE_URL` from Secrets Manager at task startup.

## Interview Explanation

You can say:

> I added RDS PostgreSQL and Secrets Manager to the ECS deployment. The database password is generated by Terraform, stored in Secrets Manager, and injected into ECS task definitions as `DATABASE_URL`. Database access is restricted to the ECS task security group.
